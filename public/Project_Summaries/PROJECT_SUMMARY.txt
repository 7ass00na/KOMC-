KOMC Law Firm Website — Project Summary (English)
=================================================

Quick Facts
- Tech: Next.js 16 (App Router), Tailwind CSS v4, Framer Motion v12, TS 5
- Node.js: 18.x LTS
- Local Dev: npm run dev → http://localhost:3000
- Prod Start: npm run start (after build)
- Editor IDs: home, about, services, cases, news (+ header/footer dedicated routes)

Overview
- Bilingual legal consultancy website (English + Arabic) with SEO-ready structure.
- Modern UX with fast performance, dark/light theme, and responsive design.
- Visual edit mode for key pages with workflow controls (draft/review/active).

Key Pages
- Home, About, Services, Cases, News (+ individual News articles)
- Team, Contact, Privacy, Terms
- Editor routes: /en/editor/[page], /ar/editor/[page] for: home, about, services, cases, news
- Editor routes also exist for header and footer: /en|ar/editor/header, /en|ar/editor/footer

Core Features
- Visual Editor
  - On-page editing for text, buttons, images, and sections.
  - Save Changes updates page-content and page-states; Reactivating exits editor.
  - “Move to Review” sets workflow status for reviews before publishing.
  - Maintenance toggle can deactivate a page and open the editor directly from Admin.
- Services
  - Service catalog with animated cards and detail panes.
  - Quick comparison matrix (desktop) and per-category lists (mobile).
- Cases
  - Cases listing with basic taxonomy awareness (tags/categories via /api/admin/taxonomy).
- News
  - Listing with taxonomy filter chips for tags/categories.
  - Detail pages under /en/news/[slug] and /ar/news/[slug].
- Team & Trust
  - Team grid with resilient images and trusted-by sections.
- Admin CMS
  - Panels for Header, Footer, Homepage, About, Services, Cases, News, Legal, Settings, Language, Colors, Workflow, Revisions.
  - Revisions: Add/Delete logs for Services/News with restore by ID (PUT/POST to endpoints).
  - Workflow: status transitions to “review”.
  - Language & Theme: toggle default language, fonts, sizes; dark mode via class.

Technology Stack
- Framework: Next.js 16 (App Router)
- Styling: Tailwind CSS v4 (dark mode via class)
- Animations: Framer Motion v12
- Language & Theme State: React Context (src/context/LanguageContext, ThemeContext)
- TypeScript 5, React 19, ESLint
- Next Features: React Compiler enabled, Next Image with remote patterns
- APIs & Data: File-backed JSON under src/data (e.g., about.json, footer.json, taxonomy.json, settings.json)
- Admin APIs:
  - /api/admin/[entity], /api/admin/page-content, /api/admin/page-states, /api/admin/auth/login
  - Also: /api/contact, /api/lead-intake, /api/upload

SEO & i18n
- Per-locale routes under /en and /ar.
- Alternates/hreflang set in metadata for main pages.
- Clean URLs for articles and static pages; image domains whitelisted in Next config.

How to Develop & Run
- Dev:       npm run dev        → http://localhost:3000
- Build:     npm run build
- Start (prod): npm run start   → http://localhost:3000
- Lint:      npm run lint -- --fix ./src

Environment Variables
- Build/Runtime
  - NODE_ENV=production (prod)
  - NEXT_TELEMETRY_DISABLED=1 (optional)
  - NEXT_PUBLIC_ASSET_VERSION=2026-03 (optional; cache-bust images/assets)
- Email (Contact API)
  - SMTP_HOST, SMTP_PORT (e.g., 587), SMTP_USER, SMTP_PASS
  - SMTP_FROM (optional; defaults to SMTP_USER)
  - If SMTP is not configured, emails are logged to server output.

Data Storage & Backup
- Location: src/data/*.json managed by lib/store with atomic writes (.tmp → rename).
- Key files:
  - about.json, footer.json, settings.json, taxonomy.json, users.json, page-states.json, page-content.json, etc.
- Backup tip: keep regular copies of src/data in versioned archives; restore by replacing JSON and restarting.

API Endpoints Reference
- Admin (file-backed JSON)
  - GET/PUT /api/admin/page-content       → per-page edit changes
  - GET/PUT /api/admin/page-states        → active/draft/workflow/schedule
  - GET/PUT /api/admin/[entity]           → about, header, footer, theme, settings, seo, scripts, robots, sitemap, taxonomy
  - POST/PUT/DELETE /api/admin/services   → services list
  - POST/PUT/DELETE /api/admin/news       → news list
  - POST /api/admin/auth/login            → default admin bootstrap and login
- Public/Utility
  - POST /api/contact                     → sends email via SMTP
  - POST /api/lead-intake                 → leads endpoint (if used)
  - POST /api/upload                      → saves to /public/uploads and returns URLs

Editor Workflow
- States per page in page-states: active, draft, workflow.status (published/review).
- “Deactivate & Open Editor” (Admin) sets draft and opens /{locale}/editor/{page}?editor=1.
- “Save Changes” writes page-content and toggles draft/active; “Move to Review” sets workflow.status=review.
- Revisions for Services/News log add/delete with payload snapshots; restore by ID via PUT/POST to endpoints.

Troubleshooting
- 404 on editor routes: ensure locale prefix (/en or /ar) and supported IDs (home/about/services/cases/news).
- Port in use: stop the process using 3000, then npm run start.
- Images blocked: extend images.remotePatterns in next.config.ts for new domains.
- Emails not sent: set SMTP_* env vars; otherwise contact API logs email to server.
- Build issues: run npm run lint, ensure Node 18.x, then npm run build.

Deployment on cPanel (English)
------------------------------
- Requirements
  - cPanel with “Setup Node.js App” (CloudLinux Node.js Selector) enabled
  - Node.js 18.x, domain/subdomain pointing to the server
- Steps (Build on Server via Node.js App)
  1) Zip and upload the project to a folder, e.g., ~/komc-site (exclude node_modules and .next).
  2) In cPanel → Setup Node.js App → Create Application:
     - Application Root: /home/USERNAME/komc-site
     - Application URL: https://your-domain.com (or a subdomain)
     - Node.js Version: 18.x
     - Application Startup File: server.js
  3) In the project root, create server.js with:
     const { createServer } = require("http");
     const next = require("next");
     const app = next({ dev: false });
     const handle = app.getRequestHandler();
     const port = process.env.PORT || 3000;
     app.prepare().then(() => {
       createServer((req, res) => handle(req, res)).listen(port, () => {
         console.log("Next.js running on", port);
       });
     });
  4) Click “Create” (or “Update”), then “Run NPM Install” to install dependencies.
  5) Build: click “Run Script” → enter build → Run, or use Terminal: npm run build.
  6) Environment variables:
     - NODE_ENV=production
     - NEXT_TELEMETRY_DISABLED=1 (optional)
  7) Click “Start Application”. Open your domain and verify.
- Alternative (Standalone, optional)
  - You may set Next.js to output standalone and rebuild, then keep server.js as above. This can reduce runtime dependencies.

Deployment on Vercel (English)
------------------------------
- Requirements
  - Vercel account and a connected Git provider (GitHub/GitLab/Bitbucket)
  - This repository pushed to your Git provider
- Steps (Dashboard)
  1) Push your code to a new repository.
  2) In Vercel, click “New Project” → Import the repository.
  3) Vercel auto-detects Next.js:
     - Build Command: next build (default)
     - Install Command: npm install (default)
     - Output: handled by Vercel’s Next.js runtime
  4) Set Environment Variables (if any) under Settings → Environment Variables.
  5) Click “Deploy”. The first build creates a preview URL; promote to Production if desired.
  6) Add Custom Domain under Settings → Domains and configure DNS (CNAME to cname.vercel-dns.com).
- CLI Alternative
  - npm i -g vercel
  - vercel (link project) → vercel --prod (production deploy)

Deployment on Netlify (English) — optional
------------------------------------------
- Steps
  1) Create a new site from Git, select your repository.
  2) Set Node version to 18+ in Site Settings → Build & Deploy → Environment.
  3) Build command: npm run build
  4) Publish directory: .next
  5) Netlify automatically uses its Next.js runtime; if prompted, keep defaults.
  6) Add environment variables as needed; deploy.

Deployment on Render (English) — optional
-----------------------------------------
- Steps
  1) Create a new Web Service from your repository.
  2) Select Node 18 LTS.
  3) Build command: npm install && npm run build
  4) Start command: npm run start
  5) Add environment variables if needed; deploy.

  7) اضغط “Start Application”، ثم افتح النطاق للتحقق.
- بديل (Standalone، اختياري)
  - يمكن ضبط Next.js على مخرج standalone ثم إعادة البناء، مع الاحتفاظ بـ server.js كما هو، لتقليل الاعتمادات وقت التشغيل.

النشر على Vercel (العربية)
-------------------------
- المتطلبات
  - حساب Vercel وربط مع GitHub/GitLab/Bitbucket
  - رفع الكود إلى المستودع
- الخطوات (عبر لوحة التحكم)
  1) ادفع الشفرة إلى مستودع جديد.
  2) في Vercel اختر “New Project” ثم استورد المستودع.
  3) يكتشف Vercel إطار Next.js تلقائيًا:
     - أمر البناء: next build (افتراضي)
     - أمر التثبيت: npm install (افتراضي)
     - المخرجات: تُدار بواسطة Runtime الخاص بـ Vercel
  4) أضف المتغيرات البيئية (إن وجدت) من Settings → Environment Variables.
  5) اضغط “Deploy”. ستحصل على رابط معاينة؛ يمكنك ترقية النشر إلى الإنتاج.
  6) أضف نطاقًا مخصصًا من Settings → Domains واضبط DNS (CNAME إلى cname.vercel-dns.com).
- عبر سطر الأوامر
  - npm i -g vercel
  - vercel (لربط المشروع) → vercel --prod (لنشر الإنتاج)

النشر على Netlify (العربية) — اختياري
------------------------------------
- الخطوات
  1) أنشئ موقعًا جديدًا من Git واختر المستودع.
  2) اضبط نسخة Node إلى 18+ من Site Settings → Build & Deploy → Environment.
  3) أمر البناء: npm run build
  4) مجلد النشر: .next
  5) يعتمد Netlify على Runtime خاص بـ Next.js تلقائيًا؛ اقبل الإعدادات الافتراضية.
  6) أضف المتغيرات البيئية إذا لزم، ثم انشر.

النشر على Render (العربية) — اختياري
------------------------------------
- الخطوات
  1) أنشئ خدمة ويب جديدة من المستودع.
  2) اختر Node 18 LTS.
  3) أمر البناء: npm install && npm run build
  4) أمر التشغيل: npm run start
  5) أضف المتغيرات البيئية إذا لزم؛ ثم انشر.
